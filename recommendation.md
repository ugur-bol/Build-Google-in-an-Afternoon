# ANTIGRAVITY — Production Roadmap (2 paragraphs)

ANTIGRAVITY is intentionally a single-binary, localhost-first crawler + real-time search engine built for the assignment constraints. To take it toward production, the first step is to separate concerns into independently scalable components: a **URL frontier** backed by a durable work queue (e.g., Redis/RabbitMQ/Kafka), a fleet of stateless crawler workers with per-host politeness (robots.txt, domain-based rate limiting), and a storage layer that persists both page fetch results and extracted terms. The in-memory inverted index would be replaced with a **durable index** (custom sharded index or a managed search platform) and a metadata store, plus snapshot/restore for recovery and reproducible re-indexing.

Operationally, the production path should add observability and reliability guarantees: structured logs, metrics and alerting (throughput, error rates, queue depth, index lag), tracing across fetch → parse → index, and guarded rollouts. Ranking can evolve from a quiz-friendly heuristic to TF‑IDF/BM25 plus document-quality signals, while query capabilities expand to multi-term queries, phrase matching, and typo tolerance. Finally, add safety controls (content-type filtering, size caps, deduplication via hashing/SimHash, and abuse protection) so the system remains predictable and respectful as crawl scale increases.
